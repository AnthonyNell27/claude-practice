# Project Rules (practice)

These are the rules Claude Code must follow when working in this folder.

## Code style
- Use 4 spaces for indentation. Never use tabs.
- Every function must have a docstring describing what it does.

## Safety
- Never delete `greetings.py`.
- Do not add code that reads or writes files on disk.
- Do not add code that connects to the internet or a database.

## Workflow
- Always propose a change and wait for approval before editing any file.
